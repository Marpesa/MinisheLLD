					Norme du MinisheLLD


-Tous les noms de fonction et les commentaires en anglais
-Une seule fonction non static par fichier
-Chaque fonction doit avoir un commentaire
/*
* brief:
*
* in:
*
* out:
*/

-Les abreviations se font en trois lettres: liste → lst, count→ cnt
-Il faut decouper les fonctions le plus possible et ne pas hesiter a utiliser des statics
-Nos commits doivent etre expilcites
-Il faut avoir une bonne arborescence de fichier

Objectifs: 
	-Le MinisheLLD doit etre assez propre et robuste pour etre poste en public sur github 🕶
	-Le MinisheLLD doit passer du premier coup “La victoire ou la mort”
